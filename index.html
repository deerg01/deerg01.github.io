Challenge 6 – Untitled(macOS) *In progress

Stack
Algorithms AIML PDF Parsing Tokenization Classification Table Data Processing macOS
Overview

In film production, converting scenario(text) into structured scene lists (tabular data) has been mostly manual. This project aims to automate that process — letting users upload a scenario PDF and receive a draft scene-list matching a common Excel format through keyword classification and context inference.

Key Challenges
  • Parser design for diverse formats: handle multiple header patterns and textual forms (directions, dialogue, narration) robustly.
  • Keyword labeling algorithm: extract and tag elements such as locations, character names, and time cues without relying on AI models.
  • AI-based inference: provide summaries and infer contextual data like time of day (day/night) to enrich scene metadata.
Approach
  • Modular parser: PDFKit extraction with structural heuristics; Vision-based OCR fallback for embedded fonts.
  • Token classification: regex/FSM-based keyword detection refined by context rules.
  • AI inference layer: lightweight transformer summarizer and time inference using contextual cues.
Outcome(In progress)
  • In progress: core parser and keyword labeling stable; AI summarization prototype under testing.

Challenge 4 – Rootrip (iOS)

Git Repository ↗
Stack
Swift SwiftUI MapKit PencilKit Firebase/Firestore SwiftData Git
Overview

Rootrip unifies trip planning in one app — map exploration, route lookup, itinerary creation, and real-time collaboration. It began as an effort to define missing capabilities, extend Apple frameworks, and build a coherent system, rather than simply assembling features.

Key Challenges
  • MapKit overlay limits: no native per-segment styling or live updates on edited paths.
  • PencilKit bitmap output: freehand strokes lacked closed-area recognition and vector scalability.
  • Data consistency: multi-user planning required local (SwiftData) ↔ cloud (Firestore) sync with conflict policy.
Approach
  • Repository pattern: decoupled view models from storage; local-first PKStroke in SwiftData, commit at PKDrawing; itineraries sync instantly to Firestore.
  • Custom path rendering: subclassed MKOverlayRenderer and overrode draw (CoreGraphics) for per-segment color/width with zoom-safe AA/scale.
  • Closed-area detection: vectorized PKStrokePoint, ray-casting polygon close, area thresholding, and interpolated boundaries on zoom.
  • Overlay orchestration: OverlayManager to prioritize annotations/drawings/routes with show/hide and state-based updates.
Outcome
  • Delivered a multi-module, maintainable codebase and a responsive, layered map UI.
  • Evolved from app assembly to framework extension, adding the pieces the SDK lacked.
